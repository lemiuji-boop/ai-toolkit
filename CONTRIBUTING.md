# Contributing

Keep changes scoped to one project unless they improve shared documentation or automation.

Before opening a pull request:

1. Read the selected project README and use its documented toolchain.
2. Add or update tests for behavioral changes.
3. Run the project formatter, linter, and tests.
4. Do not commit `.env` files, credentials, databases, generated content, dependencies, or build outputs.
5. Keep Apache 2.0 headers on source files.
