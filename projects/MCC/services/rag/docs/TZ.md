# ТЗ · СЕРВИС-05 «RAG-сервис нормативов»

Скаффолд по шаблонам документа 01. Поиск по действующей нормативной базе
(ГОСТ/ОСТ/ТУ) с обязательной привязкой к источнику. Бизнес-логики расчёта нет.

## 1. Общие сведения
| Поле | Значение |
|---|---|
| Название | RAG-сервис нормативов |
| Версия / дата | 0.1.0 / 2026 |
| Платформа | Ubuntu, Python 3.12, FastAPI, Docker; ChromaDB |
| Описание | Индексирует нормативы и отдаёт релевантные фрагменты с цитированием источника |

## 2. Цель и контекст
- **Проблема.** Коэффициенты и нормы должны опираться на действующие документы с прослеживаемым источником.
- **Бизнес-цель.** Любой коэффициент в расчёте имеет ссылку на норматив (ГОСТ/ОСТ/ТУ, пункт, версия).
- **Вне рамок.** Сервис не считает нормы (это СЕРВИС-07) и не извлекает данные из КД.

## 3. Пользователи и сценарии
- **Роли.** Локальный оркестратор / агент верификации (запрашивают нормативы).
- **US-1.** Найти норматив по запросу и получить источник. → FR-002, FR-003
- **US-2.** Пополнить индекс новыми нормативами. → FR-001

## 6. Функциональные требования (FR)
| ID | Требование | Критерий приёмки | Приоритет |
|---|---|---|---|
| FR-001 | Индексация нормативов (`POST /index`) | Документы добавляются; `total` растёт | MUST |
| FR-002 | Семантический поиск (`POST /search`) | Возвращает ранжированные hits по запросу | MUST |
| FR-003 | Цитирование источника | У каждого hit есть непустое поле `source` | MUST |
| FR-004 | `/health`, `/version` | 200; `/version` отдаёт версию и бэкенд хранилища | MUST |

## 9. Настройки
`CHROMA_PATH` (путь persistent-хранилища; пусто → in-memory), `COLLECTION`, `EMBED_DIM`, `TOP_K`.

## 10. Интеграции
Вызывается из локального контура (оркестратор/верификация). Внешние API запрещены.
Эмбеддинги детерминированные (хеширование), без загрузки моделей из сети.

## 12. Безопасность
- Нет исходящих обращений во внешнюю сеть.
- Нормативная база хранится локально (volume `rag_data`).

## 13. Архитектура и стек
- **Стек.** Python 3.12, FastAPI, Pydantic v2, ChromaDB; pytest/ruff/mypy.
- **Структура.** `app/main.py` (роуты), `app/core` (config, schemas, embed), `app/store.py` (бэкенд хранилища), `app/seed.py` (демо-нормативы).
- **ADR.** ChromaDB как векторное хранилище; эмбеддинги передаются явно, поэтому модель эмбеддингов не загружается (on-premise). Продуктивная модель (bge-m3) подключается без смены интерфейса `embed.embed`.

## 14. Открытые вопросы
- Источник и формат нормативной базы предприятия (загрузка/обновление индекса).
- Выбор продуктивной модели эмбеддингов (bge-m3 локально).

## 15. Definition of Done
- [ ] `/index` и `/search` работают; у hits есть `source`;
- [ ] `/health`, `/version` отвечают;
- [ ] `ruff`/`mypy`/`pytest` зелёные;
- [ ] сервис собирается и запускается в контейнере.
