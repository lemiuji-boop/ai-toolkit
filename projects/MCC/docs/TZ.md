# ТЗ · МАТНОРМ backend (монолитный каркас, vertical slice)

Адаптация ТЗ из документа 02 (СЕРВИС-02 Vision-LLM) и декомпозиции документа 01
под единый FastAPI-бэкенд, где каждый модуль `app/services/*` соответствует
будущему отдельному сервису и заменяется на полный без изменения публичного API.

## 1. Общие сведения

| Поле | Значение |
|---|---|
| Название | МАТНОРМ backend (каркас расчёта норм расхода материалов) |
| Версия / дата | 0.1.0 / 2026 |
| Платформа | Ubuntu, Python 3.12, FastAPI, Docker; GPU-узел для inference |
| Описание | Принимает чертёж (скан) и 3D-модель, извлекает поля, сверяет, считает нормы расхода, отдаёт ведомость в Excel-надстройку |

## 2. Цель и контекст
- **Проблема.** Чертежи поступают сканами без текстового слоя; ручной расчёт норм медленный и подвержен ошибкам.
- **Бизнес-цель.** Сквозной автоматический конвейер «КД + 3D → нормы расхода» с перекрёстной верификацией.
- **Вне рамок.** Дообучение модели, NX/CATIA native, шлюз-страж, внешний оркестратор, Kafka — отдельные сервисы (см. §14).

## 3. Пользователи и сценарии
- **Роли.** Технолог-нормировщик (через Excel-надстройку); локальный оркестратор (в полной системе).
- **User Stories.**
  - US-1. Отправить чертёж + 3D и получить ведомость норм. → FR-001, FR-010, FR-011
  - US-2. Видеть уверенность по полям распознавания. → FR-003
  - US-3. Видеть флаги верификации и нормоконтроля. → FR-012, FR-013

## 4. Глоссарий
Основная надпись — штамп по ГОСТ 2.104. Мд/Мз — масса детали / заготовки. КИМ — коэффициент использования материала (Мд/Мз). Входимость — количество детали на 1 самолёто-комплект (с/к). VLM — vision-language model.

## 5. Целевые платформы
Сервис без UI (REST). UI — Excel-надстройка (`addin/`), описана отдельно.

## 6. Функциональные требования (FR)

| ID | Требование | Критерий приёмки | Приоритет | US |
|---|---|---|---|---|
| FR-001 | `POST /api/jobs` принимает чертёж (+опц. 3D) и возвращает `JobResult` | Валидный по схеме JSON; нет чертежа → ошибка | MUST | US-1 |
| FR-002 | OCR-препроцесс: локализовать зоны (основная надпись, спецификация) до VLM; PDF→PNG | Зоны выделены; координаты доступны в debug | MUST | US-1 |
| FR-003 | По каждому непустому полю — оценка уверенности [0..1] | `confidence` присутствует для непустых полей | MUST | US-2 |
| FR-004 | Нераспознанные поля → null, не выдумывать | Доля «выдуманных» при нечитаемом поле ≤ 2 % | MUST | US-1 |
| FR-007 | `/health` и `/version` | GET /health → 200; /version отдаёт версию сервиса и модели | MUST | — |
| FR-010 | Геометрия из 3D (STEP): объём, габариты, масса, дерево сборки | На STEP считаются объём/габариты; для сборки — дерево с кратностями | MUST | US-1 |
| FR-011 | Расчёт норм по правилам `rules.json` (детерминированно, без LLM) | `NormRow` с Мд/Мз/КИМ/нормами; правила меняются в JSON | MUST | US-1 |
| FR-012 | Перекрёстная сверка чертёж ↔ 3D (3D — эталон размеров) | Расхождение габаритов/массы → флаг | MUST | US-3 |
| FR-013 | Нормоконтроль строк ведомости (`/api/normcontrol`) | Нарушения порогов КИМ и отсутствие марки → флаги | MUST | US-3 |
| FR-014 | Входимость: qty на с/к = произведение кратностей по дереву × `SETS_PER_AIRCRAFT` | На заглушке-сборке qty считается верно | MUST | US-1 |
| FR-015 | Виды заготовки (лист/профиль/поковка/литьё) с припусками из `rules.json` | На каждый вид — свой коэффициент; выбор переопределяем в JSON | SHOULD | US-1 |

> Правило полноты: каждый FR сопоставляется с кодом и тестом в `docs/trace.md`.

## 7. Нефункциональные требования (NFR)

| ID | Категория | Требование | Целевое значение |
|---|---|---|---|
| NFR-001 | Производительность | Извлечение одной страницы (GPU) | ≤ 20 с |
| NFR-002 | Стабильность | Деградация к заглушке при недоступном inference/CAD | Каркас не падает, `source=stub` |
| NFR-003 | Локализация | Кириллица в распознавании и расчётах | Поддержана |
| NFR-004 | Ресурсы | VRAM пилота (RTX 3060) | ≤ 6 ГБ (модель 3B/4B/8B-Q4) |

## 8. Контракт API
- `GET /health` → `{status: "ok"}`
- `GET /version` → `{service, version, model}`
- `POST /api/jobs` (multipart: `drawing`, опц. `model3d`; query: `debug`) → `JobResult{extract, geometry, verify, rows}`
- `POST /api/normcontrol` (`{rows}`) → `{flags, rows}`
- `POST /api/excel/export` (`{rows}`) → xlsx

## 9. Настройки (переменные окружения)
`INFERENCE_URL`, `VLM_MODEL`, `INFERENCE_TIMEOUT`, `IMAGE_DPI`, `OCR_ENABLED`, `CONF_THRESHOLD`, `SETS_PER_AIRCRAFT`, `CORS_ORIGINS`.

## 10. Интеграции
Локальный inference (Ollama) по `INFERENCE_URL`. RAG-сервис нормативов (СЕРВИС-05) — отдельный контейнер. Внешние API запрещены (on-premise).

## 11. Данные
Сервис stateless. Исходные изображения не хранятся. Обозначения изделий конфиденциальны — не логируются в открытом виде. ПДн отсутствуют.

## 12. Безопасность (SEC)

| ID | Требование | Критерий приёмки |
|---|---|---|
| SEC-001 | Нет обращений во внешнюю сеть из локального контура | Сетевые политики запрещают egress |
| SEC-002 | Обозначения изделий не попадают в логи открыто | Аудит логов — только хеши/идентификаторы |
| SEC-003 | Секреты только через переменные окружения | В коде/репозитории секретов нет |
| SEC-004 | Зависимости с фиксированными версиями | `pyproject.toml` фиксирует версии |

## 13. Архитектура и стек
- **Стек.** Python 3.12, FastAPI, Pydantic v2, pydantic-settings, httpx, openpyxl, pymupdf, cadquery (extra `cad`); pytest/ruff/mypy.
- **Структура.** `app/api` (роуты), `app/core` (схемы, конфиг), `app/services` (vision, ocr, geometry, verify, calc, excel_format), `app/data/rules.json`, `tests/`.
- **Точки расширения (без слома API):** `vision.py` (дообученная Qwen3-VL + OCR), `geometry.py` (NX/CATIA, дерево сборки), `calc.py` + `rules.json` (полные правила), новый `services/rag/` (СЕРВИС-05).
- **Правила агента.** Источник правды — это ТЗ; LLM не считает нормы; кириллица критична; on-premise.

## 14. Ограничения, допущения, открытые вопросы
- **Ограничения.** Продуктив на GPU ≥ 16 ГБ; пилот на RTX 3060 6 ГБ — модель 3B/4B/8B-Q4.
- **Откладывается (вне scope каркаса):** СЕРВИС-01 (оркестратор-автомат), СЕРВИС-08 (шлюз-страж), СЕРВИС-09 (внешний оркестратор), СЕРВИС-11 (Kafka/downstream), дообучение Qwen3-VL (этапы О-1…О-6), NX/CATIA native.
- **Открытые вопросы.** (1) Внутренний TLS для устранения mixed content (Vercel HTTPS → HTTP backend) — отв. сетевой администратор. (2) Допустимость Vercel — согласовать с ИБ. (3) Кириллица в OCR выбранной модели — отв. ML-инженер. (4) Админ-панель мониторинга (`/api/admin/*`, UI на порту 3010): аутентификация/RBAC и сетевой доступ (только внутренняя сеть vs VPN) — согласовать с ИБ; в каркасе панель без auth для локальной отладки. (5) **Cloudflare quick tunnel** для Excel-надстройки: egress только на уровне хоста (`cloudflared`), не из кода бэкенда (SEC-001); допустимость временного публичного URL для КД — согласовать с ИБ (конфиденциальность SEC-002, mixed content).
