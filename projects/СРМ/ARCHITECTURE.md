# Архитектура каркаса и открытые вопросы

## Поток
Excel (надстройка с Vercel-оболочкой) → POST /api/jobs на Ubuntu-бэкенд →
vision.extract (LLM/заглушка) → geometry (STEP/заглушка) → verify (сверка) →
calc.build_rows (правила rules.json) → строки ведомости → надстройка пишет в лист
и вызывает /api/normcontrol для подсветки замечаний.

## Точки расширения (замена без изменения API)
- `services/vision.py` — подключить дообученную Qwen3-VL и OCR-препроцессор.
- `services/geometry.py` — добавить NXOpen/CATIA, дерево сборки и входимость.
- `services/calc.py` + `data/rules.json` — полные нормативные правила и коэффициенты.
- Добавить сервисы: RAG (05), шлюз-страж (08), downstream-API (11).

## Открытые вопросы (решить со службой ИБ до продуктива)
1. **Внешний HTTPS бэкенда.** Vercel-оболочка отдаётся по HTTPS; браузер
   заблокирует вызовы к HTTP-бэкенду (mixed content). Нужен внутренний TLS
   (nginx + корпоративный сертификат) или доступ по https к Ubuntu-хосту.
2. **Доступность хоста с ПК пользователей.** Внутренний DNS/маршрут до Ubuntu-хоста
   (VPN/корпоративная сеть). Отв.: сетевой администратор.
3. **Допустимость Vercel.** Согласовать, что облако хостит только UI-код без данных;
   при запрете — заменить Vercel на внутренний nginx (та же статика), бэкенд не меняется.
4. **CORS.** В проде сузить `cors_origins` до точного URL надстройки.
